# Zenn 公開前チェック

CLAUDE.md の「記事執筆のルール」への **適合** を機械的に判定する。判断ベースの編集レビュー（流れ、論旨、文体の読みやすさなど）は `zenn-content-reviewer` エージェントの責務。

## 対象ファイルの特定

ユーザーが対象記事を明示していなければ、`articles/` 内のファイル一覧を提示して選んでもらう。

## チェック項目

### 1. frontmatter

CLAUDE.md の frontmatter 表の制約を満たしているか確認する。加えて：

- `published` が `false` であること（このスキルでは `true` にしない）

### 2. slug（ファイル名）

CLAUDE.md の slug ルールに準拠しているか確認する。

### 3. 本文構造

- 本文の先頭が `##` から始まる（`#` を使っていない）
- 見出しレベルが飛んでいない（`##` の次に `####` など）
- HTML コメント（`<!-- ... -->`）が残っていない（骨組みのメモは公開前に削除）

### 4. コードブロック

- すべての ` ``` ` ブロックに言語指定がある
  - 言語指定なしを見つけたら、内容から推測して提案する

### 5. 画像

- すべての画像参照（`![...](...)`）について：
  - 絶対パス（`/images/...`）になっている
  - 参照先のファイルが実在する
  - alt テキストが空でない

### 6. リンク

- Markdown リンク（`[text](url)`）の text が空でない
- 明らかに壊れた URL（`http`/`https` で始まらない外部リンク）がない

### 7. 表記ルール

- 文体が「です・ます調」で統一されている
- 半角英数字の前後に半角スペースがある
- 用語の表記揺れがない（CLAUDE.md の「カタカナ語・固有名詞」セクション参照）

## アウトプット形式

```
## 公開前チェック結果

### ❌ 修正が必要
- [箇所] [問題] → [修正案]

### ✅ 問題なし
- frontmatter / slug / 構造 / コードブロック / 画像 / リンク / 表記
```

問題があれば箇所と修正案を、無ければ ✅ のみ報告する。

## 修正の進め方

- 機械的に修正できるもの（表記揺れ、半角スペース、コードブロックの言語指定など）は、ユーザー承認後に一括修正する
- 修正後は再度このスキルを実行することを推奨する
- 公開前は機械チェック後に **`zenn-content-reviewer` の編集レビューも併用** することを案内する

## 重要な制約

- このスキル単体では `published: true` に変更しない（公開は `/publish-article` で行う）
- ユーザー承認なしに本文の構造を大きく変えない
- 文章の質や流れに関する判断はこのスキルの責務外（`zenn-content-reviewer` に委ねる）
